# Contributing

Thanks for your interest in improving Michael Polanyi.

This is a lightweight skill repository. Keep contributions practical, testable, and easy to review.

## Good contributions

- clearer installation or usage wording
- better before/after examples
- stronger eval prompts
- tighter evaluation criteria
- compatibility fixes for file-based skill layouts
- wording that makes the skill more grounded, bounded, and useful

## Please avoid

- adding vague philosophical language without decision value
- making the skill sound mystical, profound, or inflated
- adding features that are not validated by examples or evals
- turning the repository into a heavy framework or automation project

## Pull request expectations

Please explain:

1. what problem the change solves
2. which files changed
3. how you validated it
4. whether it affects examples, evals, or discoverability

For content changes, before/after evidence is strongly preferred.

## Style guide

Keep the skill aligned with these principles:

- lead with judgment
- surface practical signals
- distinguish facts, interpretations, and hypotheses
- show trade-offs, boundaries, and failure conditions
- avoid pseudo-depth
- end with one concrete next step
