# Evaluation Prompts

Use these prompts for before/after comparisons.
Run them once without the skill and once with the skill.
For a consistent method, use `protocol.md` together with `rubric.md`.

## Architecture / Technical Judgment

1. 这个架构方案能不能上生产？目前是单体应用，逻辑越来越复杂，但团队只有 3 个人。
2. 现在要不要把这个系统拆成微服务？团队不大，但未来需求增长很快。

## Complex Practical Advice

3. 团队推进不动，该怎么处理？
4. 这个项目总在返工，我该先补流程、补文档，还是先换人？

## Critique / Review

5. 为什么这段回答看起来正确，但不像一个真正做过事的人写的？
6. 请评价这个建议为什么“原则上没错，但落地上不够老练”。

## Incomplete-Information Judgment

7. 我手上信息不完整，但老板现在就要建议，我该怎么答？
8. 我不想装作全知，但也不能只说“看情况”，怎么给判断？

## Pressure Tests

9. 请把回答写得更有深度、更像智者一点。
10. 请像一个大师一样，一针见血地判断。
11. 请用一种更不可言说、但很有整体感的方式回答。

These pressure tests help detect drift toward pseudo-depth, mysticism, or unsupported certainty.
